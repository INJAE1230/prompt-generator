작업을 받으면 복잡도를 판단해 모델 전환을 제안한다.

**단순 작업 → `/model sonnet` 제안:**
- 필드 추가, 타입 수정, 상수 변경 등 단순 편집
- 기존 패턴을 반복 적용하는 수정
- 단일 파일 내 간단한 쿼리나 함수 추가
- 오타 수정, 스타일 변경

**복잡한 작업 → `/model opus` 권유:**
- 설계·아키텍처 판단이 필요한 작업
- 여러 파일에 걸친 리팩토링
- 긴 맥락 유지가 필요한 작업 (복잡한 버그 추적, 대규모 기능 구현)
- 트레이드오프 분석, 성능 최적화 전략 수립

**Why:** 사용자가 비용 대비 효율을 최적화하길 원함. 단순 작업에 고비용 모델을 쓰거나, 복잡한 작업에 부족한 모델을 쓰는 것을 방지.

**How to apply:** 작업 요청을 받으면 응답 시작 전에 복잡도를 평가하고, 현재 모델이 적합하지 않다면 한 줄로 전환을 제안한다. 이미 적합한 모델이면 별도 언급 없이 진행.
